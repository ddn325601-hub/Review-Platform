# 学习路线

建议采用“双轨复现”：左边用 `backend/hm-dianping-starter` 按讲义手写，右边用 `backend/hm-dianping-complete` 对照验证。这样不会被完整代码牵着走，也不会在卡住时没参照。

## 第 0 步：跑通基础项目

目标：

- MySQL 导入 `hmdp.sql`
- Redis 可连接
- 后端 `8081` 启动成功
- 前端页面能请求 `/api`

对应文档：`docs/本地复现指南.md`

## 第 1 步：短信登录与 Token

核心问题：

- 为什么不能依赖单机 `Session`
- 验证码为什么适合放 Redis 并设置 TTL
- 登录用户为什么用 token hash 存储
- 拦截器如何刷新 token 有效期

重点代码：

- `UserServiceImpl#sendCode`
- `UserServiceImpl#login`
- `RefreshTokenInterceptor`
- `LoginInterceptor`
- `MvcConfig`

## 第 2 步：商户缓存

核心问题：

- 缓存穿透、击穿、雪崩分别是什么
- 空值缓存、互斥锁、逻辑过期分别解决什么问题
- 更新数据库后为什么删除缓存

重点代码：

- `ShopServiceImpl#queryById`
- `ShopServiceImpl#update`
- `CacheClient`
- `RedisData`

## 第 3 步：优惠券秒杀

核心问题：

- 为什么库存判断和一人一单要交给 Lua 保证原子性
- 为什么订单落库要异步化
- Redis Stream 的 pending list 解决什么问题
- Redisson 锁和手写锁的区别

重点代码：

- `VoucherOrderServiceImpl#seckillVoucher`
- `VoucherOrderServiceImpl.VoucherOrderHandler`
- `seckill.lua`
- `SimpleRedisLock`
- `RedissonConfig`

## 第 4 步：附近商户 GEO

核心问题：

- 店铺坐标如何按类型写入 Redis GEO
- 附近查询如何兼顾距离排序和分页
- 为什么数据库二次查询要保持 Redis 返回的顺序

重点代码：

- `HmDianPingApplicationTests#loadShopData`
- `ShopServiceImpl#queryShopByType`

## 第 5 步：签到与 UV

核心问题：

- BitMap 如何表示一个月的连续签到
- `BITFIELD` 取出的数字如何计算连续签到天数
- HyperLogLog 为什么适合 UV 估算

重点代码：

- `UserServiceImpl#sign`
- `UserServiceImpl#signCount`
- `HmDianPingApplicationTests#testHyperLogLog`

## 第 6 步：好友关注与 Feed 流

核心问题：

- Set 如何做共同关注
- ZSet 如何做点赞排行榜
- Feed 流为什么用滚动分页而不是普通页码分页

重点代码：

- `FollowServiceImpl`
- `BlogServiceImpl#likeBlog`
- `BlogServiceImpl#queryBlogLikes`
- `BlogServiceImpl#saveBlog`
- `BlogServiceImpl#queryBlogOfFollow`
