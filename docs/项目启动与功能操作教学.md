# 项目启动与功能操作教学

这份文档面向第一次运行和学习黑马点评项目的过程，包含启动、验证、页面操作、接口验证、Redis 观察点和常见问题处理。

## 1. 项目结构

| 路径 | 说明 |
| --- | --- |
| `backend/hm-dianping-starter` | 起始版后端，推荐学习时从这里动手写 |
| `backend/hm-dianping-complete` | 完整实现版后端，用来对照和验证 |
| `frontend/hmdp` | 前端静态页面 |
| `docs` | 讲义、学习路线、接口清单、启动指南 |
| `resources/db/hmdp.sql` | MySQL 初始化数据 |
| `resources/jmeter/秒杀抢购.jmx` | 秒杀压测脚本 |
| `scripts` | 本地启动和停止脚本 |

## 3. 一键启动和停止

在项目根目录执行：

```powershell
.\scripts\start-starter.ps1
```

启动后访问：

```text
http://127.0.0.1:8082
```

停止本项目启动的服务：

```powershell
.\scripts\stop-local.ps1
```

## 4. 手动启动后端

起始版：

```powershell
cd backend\hm-dianping-starter
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

完整实现版：

```powershell
cd backend\hm-dianping-complete
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

如果只是学习前几章，建议先运行起始版。完整实现版包含 Redis Stream 秒杀队列逻辑，当前 Windows Redis `3.2.100` 不支持 Stream；要完整跑秒杀异步下单，需要 Redis 5+。

## 5. 启动后验证

验证后端：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8081/shop-type/list
```

验证前端代理：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8082/api/shop-type/list
```

返回内容中出现：

```json
{"success":true}
```

说明前端、Nginx、后端、MySQL 链路正常。

## 6. 页面功能操作

### 6.1 首页浏览

打开：

```text
http://127.0.0.1:8082
```

可操作内容：

- 查看商户分类
- 查看热门探店笔记
- 点击商户进入详情页
- 点击底部导航切换页面

对应学习点：

- 前端通过 `/api/shop-type/list` 获取分类
- 后端查询 `tb_shop_type`
- Nginx 把 `/api` 转发到 `8081`

### 6.2 登录

进入登录页：

```text
http://127.0.0.1:8082/login.html
```

操作步骤：

1. 输入手机号，例如 `13800138000`
2. 点击发送验证码
3. 查看后端日志中的验证码
4. 输入验证码登录

后端日志：

```text
logs/backend-starter.out.log
```

对应学习点：

- `UserServiceImpl#sendCode`
- `UserServiceImpl#login`
- Redis `login:code:{phone}`
- Redis `login:token:{token}`
- 请求头 `authorization`

### 6.3 商户查询

页面操作：

- 首页点击商户分类
- 进入商户列表
- 点击商户详情

接口：

```text
GET /shop/of/type
GET /shop/{id}
```

对应学习点：

- 缓存旁路
- 缓存穿透
- 缓存击穿
- 缓存更新策略

观察 Redis key：

```text
cache:shop:{id}
```

### 6.4 探店笔记

页面操作：

- 首页查看热门探店
- 点击笔记详情
- 登录后点赞
- 登录后发布探店内容

接口：

```text
GET /blog/hot
GET /blog/{id}
PUT /blog/like/{id}
POST /blog
```

对应学习点：

- ZSet 点赞排行榜
- Feed 流推送
- 用户信息补全

观察 Redis key：

```text
blog:liked:{blogId}
feed:{userId}
```

### 6.5 关注与共同关注

页面操作：

- 进入其他用户主页
- 点击关注或取消关注
- 查看共同关注

接口：

```text
PUT /follow/{id}/{isFollow}
GET /follow/common/{id}
```

对应学习点：

- Set 保存关注关系
- Set 交集求共同关注

观察 Redis key：

```text
follows:{userId}
```

### 6.6 签到

登录后调用签到接口：

```text
POST /user/sign
GET /user/sign/count
```

对应学习点：

- BitMap 存储每天签到状态
- `BITFIELD` 统计连续签到天数

观察 Redis key：

```text
sign:{userId}:yyyyMM
```

### 6.7 优惠券秒杀

接口：

```text
POST /voucher-order/seckill/{voucherId}
```

学习顺序：

1. 普通数据库扣库存
2. 一人一单判断
3. synchronized 本地锁
4. Redis 分布式锁
5. Redisson
6. Lua 原子判断
7. Redis Stream 异步落库

当前环境提醒：

- 当前 Redis `3.2.100` 不支持 Stream。
- 可以先学习 Lua 前的业务演进。
- 完整 Stream 章节建议使用 Redis 5+ 或 Docker Redis 6.2。

## 9. 常见问题

### 9.1 打开 `8081` 是 404

正常。`8081` 是后端 API，不是页面入口。页面入口是：

```text
http://127.0.0.1:8082
```

### 9.2 页面能打开但数据加载失败

检查：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8082/api/shop-type/list
```

如果失败，继续看：

```text
logs/backend-starter.out.log
resources/runtime/nginx-dev/nginx-1.18.0/logs/error.log
```

### 9.3 后端启动失败，提示端口占用

检查：

```powershell
Get-NetTCPConnection -LocalPort 8081,8082
```

停止本项目服务：

```powershell
.\scripts\stop-local.ps1
```

### 9.4 数据库连接失败

确认配置：

```text
MySQL: 127.0.0.1:3307
user: root
password: l798267901
```

验证：

```powershell
mysql --protocol=TCP --host=127.0.0.1 --port=3307 --user=root --password=l798267901 -e "SELECT VERSION();"
```

### 9.5 Redis 连接失败

确认配置：

```text
Redis: 127.0.0.1:6380
password: l798267901
```

验证：

```powershell
D:\Program Files\Redis\redis-cli.exe -h 127.0.0.1 -p 6380 -a l798267901 PING
```

返回 `PONG` 即正常。
