# 项目启动指南

本文说明如何在本地启动 Review Platform。仓库不保存个人密码，请通过环境变量或本地忽略文件配置数据库与 Redis 连接。

## 1. 环境要求

| 组件 | 建议版本 |
| --- | --- |
| JDK | 8 或 11 |
| Maven | 3.6+ |
| MySQL | 5.7+ |
| Redis | 5+，完整秒杀队列需要 Stream |
| Nginx | 可选，用于静态资源和 `/api` 反向代理 |

## 2. 本地配置

复制示例配置：

```powershell
Copy-Item .\config\local-env.example.ps1 .\config\local-env.ps1
```

编辑 `config/local-env.ps1`：

```powershell
$env:HMDP_MYSQL_HOST = "127.0.0.1"
$env:HMDP_MYSQL_PORT = "3306"
$env:HMDP_MYSQL_DATABASE = "hmdp"
$env:HMDP_MYSQL_USERNAME = "root"
$env:HMDP_MYSQL_PASSWORD = "your_mysql_password"

$env:HMDP_REDIS_HOST = "127.0.0.1"
$env:HMDP_REDIS_PORT = "6379"
$env:HMDP_REDIS_PASSWORD = "your_redis_password"
```

在当前 PowerShell 会话加载配置：

```powershell
. .\config\local-env.ps1
```

`config/local-env.ps1` 已被 Git 忽略，适合保存本地私有配置。

## 3. 启动依赖

可使用仓库内的 Docker Compose 启动 MySQL 与 Redis：

```powershell
docker compose up -d
docker compose ps
```

首次启动 MySQL 会自动导入：

```text
resources/db/hmdp.sql
```

如果使用完整实现版，并且需要运行 Redis Stream 秒杀队列，先创建消费组：

```powershell
docker exec -it hmdp-redis redis-cli -a 123321 XGROUP CREATE stream.orders g1 0 MKSTREAM
```

提示 `BUSYGROUP` 表示消费组已存在，可以忽略。

## 4. 启动后端

启动完整实现版：

```powershell
cd backend\hm-dianping-complete
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

启动起始版：

```powershell
cd backend\hm-dianping-starter
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

默认后端端口：

```text
http://127.0.0.1:8081
```

后端是 API 服务，直接访问根路径可能没有页面，这是正常现象。

## 5. 启动前端

前端目录：

```text
frontend/hmdp
```

推荐使用 Nginx 或其他静态服务器托管该目录，并把 `/api` 代理到后端：

```text
http://127.0.0.1:8081
```

示例 Nginx 片段：

```nginx
location / {
    root   /path/to/frontend/hmdp;
    index  index.html;
}

location /api {
    rewrite /api/(.*) /$1 break;
    proxy_pass http://127.0.0.1:8081;
}
```

## 6. 快速验证

验证后端：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8081/shop-type/list
```

验证前端代理：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8082/api/shop-type/list
```

返回内容中出现 `success=true`，说明数据库、后端和代理链路正常。
