# 项目启动与功能操作指南

本文用于说明项目启动、页面入口、接口验证和主要功能操作。

## 1. 项目结构

| 路径 | 说明 |
| --- | --- |
| `backend/hm-dianping-starter` | 起始版后端 |
| `backend/hm-dianping-complete` | 完整实现版后端 |
| `frontend/hmdp` | 前端静态页面 |
| `resources/db/hmdp.sql` | MySQL 初始化数据 |
| `resources/jmeter/秒杀抢购.jmx` | 秒杀压测脚本 |
| `docs` | 启动、接口、Redis key 和代码说明 |

## 2. 启动顺序

1. 启动 MySQL 与 Redis。
2. 导入 `resources/db/hmdp.sql`。
3. 加载 `config/local-env.ps1` 中的本地连接配置。
4. 启动 `backend/hm-dianping-complete` 或 `backend/hm-dianping-starter`。
5. 启动前端静态服务，并配置 `/api` 反向代理。

后端默认端口：

```text
http://127.0.0.1:8081
```

前端建议端口：

```text
http://127.0.0.1:8082
```

`8081` 是 API 服务，不是网页入口；页面入口应访问前端服务端口。

## 3. 后端启动命令

完整实现版：

```powershell
cd backend\hm-dianping-complete
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

起始版：

```powershell
cd backend\hm-dianping-starter
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

## 4. 验证命令

后端直连：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8081/shop-type/list
```

前端代理：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8082/api/shop-type/list
```

返回 JSON 中出现 `success` 字段，说明基础链路正常。

## 5. 页面功能

### 首页

入口：

```text
http://127.0.0.1:8082
```

可操作内容：

- 查看商户分类
- 查看热门探店笔记
- 点击商户进入详情页
- 通过底部导航切换页面

相关接口：

```text
GET /shop-type/list
GET /blog/hot
GET /shop/{id}
```

### 登录

入口：

```text
http://127.0.0.1:8082/login.html
```

操作步骤：

1. 输入手机号，例如 `13800138000`。
2. 点击发送验证码。
3. 在后端日志中查看验证码。
4. 输入验证码完成登录。

相关接口：

```text
POST /user/code
POST /user/login
GET /user/me
```

相关 Redis key：

```text
login:code:{phone}
login:token:{token}
```

### 商户查询

可操作内容：

- 按分类查看商户列表
- 查看商户详情
- 更新商户后清理或重建缓存

相关接口：

```text
GET /shop/of/type
GET /shop/{id}
PUT /shop
```

相关 Redis key：

```text
cache:shop:{id}
shop:geo:{typeId}
```

### 探店笔记

可操作内容：

- 查看热门笔记
- 查看笔记详情
- 登录后点赞或取消点赞
- 登录后发布探店内容

相关接口：

```text
GET /blog/hot
GET /blog/{id}
PUT /blog/like/{id}
POST /blog
```

相关 Redis key：

```text
blog:liked:{blogId}
feed:{userId}
```

### 关注关系

可操作内容：

- 关注或取消关注用户
- 查询共同关注
- 查看关注用户发布的内容流

相关接口：

```text
PUT /follow/{id}/{isFollow}
GET /follow/common/{id}
GET /blog/of/follow
```

相关 Redis key：

```text
follows:{userId}
feed:{userId}
```

### 签到

相关接口：

```text
POST /user/sign
GET /user/sign/count
```

相关 Redis key：

```text
sign:{userId}:yyyyMM
```

### 优惠券秒杀

相关接口：

```text
POST /voucher/seckill
POST /voucher-order/seckill/{voucherId}
```

完整异步下单流程依赖 Redis Stream，需要 Redis 5+。如果使用低版本 Redis，可以先验证库存判断、一人一单、分布式锁和 Lua 原子判断等流程。

## 6. 常用排查

查看端口占用：

```powershell
Get-NetTCPConnection -LocalPort 8081,8082
```

停止本项目辅助进程：

```powershell
.\scripts\stop-local.ps1
```

测试 MySQL：

```powershell
mysql --protocol=TCP --host=127.0.0.1 --port=3306 --user=root --password=your_mysql_password -e "SELECT VERSION();"
```

测试 Redis：

```powershell
redis-cli -h 127.0.0.1 -p 6379 -a your_redis_password PING
```

如果页面能打开但数据加载失败，优先检查 `/api` 代理是否正确转发到 `http://127.0.0.1:8081`。
