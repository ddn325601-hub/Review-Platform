# 项目学习指南

## 学习原则

建议用 `backend/hm-dianping-starter` 做主线练习，用 `backend/hm-dianping-complete` 做答案对照。每个章节都按这三个动作走：

1. 先读讲义对应小节。
2. 在起始版中自己实现。
3. 和完整实现版做局部 diff，再跑接口验证。

## 推荐顺序

| 顺序 | 主题 | 目标 | 关键文件 |
| --- | --- | --- | --- |
| 1 | 项目跑通 | 前端能访问，接口能返回数据 | `项目启动指南.md` |
| 2 | 短信登录 | 验证码、token、拦截器 | `UserServiceImpl`, `MvcConfig` |
| 3 | 商户缓存 | 缓存穿透、击穿、逻辑过期 | `ShopServiceImpl`, `CacheClient` |
| 4 | 优惠券秒杀 | Lua、一人一单、分布式锁 | `VoucherOrderServiceImpl`, `seckill.lua` |
| 5 | 附近商户 | GEO 查询和距离排序 | `ShopServiceImpl#queryShopByType` |
| 6 | 签到统计 | BitMap 和连续签到 | `UserServiceImpl#signCount` |
| 7 | 关注和 Feed | Set、ZSet、滚动分页 | `FollowServiceImpl`, `BlogServiceImpl` |

## 每章复现方法

### 1. 短信登录

先实现：

- `sendCode`
- `login`
- 登录拦截器
- token 刷新

验证：

```text
POST /user/code
POST /user/login
GET /user/me
```

观察 Redis：

```text
login:code:{phone}
login:token:{token}
```

### 2. 商户缓存

先实现最简单的缓存旁路，再处理异常场景：

- 缓存命中
- 缓存穿透：空值缓存
- 缓存击穿：互斥锁
- 缓存击穿：逻辑过期
- 更新商户后删除缓存

验证：

```text
GET /shop/{id}
PUT /shop
```

### 3. 秒杀

建议等 Redis 5+ 环境准备好后再完整复现：

- 普通数据库扣库存
- synchronized 一人一单
- Redis 分布式锁
- Redisson
- Lua 原子判断
- Stream 异步落库

当前 Redis 3.2 可以先学习 Lua 前的思路；Stream 章节先读代码和文档。

### 4. GEO、BitMap、Set、ZSet

这几章适合边跑测试边观察 Redis 数据结构：

- GEO：先运行 `loadShopData`
- BitMap：调用签到接口后看 bit 位
- Set：关注、取关、共同关注
- ZSet：点赞排行、Feed 流

## 对照代码技巧

看某个文件差异：

```powershell
git diff --no-index backend\hm-dianping-starter\src\main\java\com\hmdp\service\impl\UserServiceImpl.java backend\hm-dianping-complete\src\main\java\com\hmdp\service\impl\UserServiceImpl.java
```

全项目差异：

```powershell
git diff --no-index backend\hm-dianping-starter backend\hm-dianping-complete
```

全项目差异会比较大，建议只在阶段复盘时使用。
